Unfortunately, this problem occurs not only in Docker but also in the npm version on Raspberry Pi OS.

Yes, I didn't want to change the title.
History: Initially this was only a problem in Docker (which was introduced because running headless locally was a problem (and which I don't use myself, at least not for this)). With node it was working for me, gave up trying to fix it in Docker, stopped following this. At some point captchas were also a problem with node (due to playwright leaking). This leaking of information that can be detected as automation is patched by patchright and rebrowser-patches. Only tried patchright some weeks ago, but basic stuff didn't work - maybe I tried it at the wrong time, it seems better now, but still has some issues.

Now with patchright, I don't get captchas for epic-games anymore.
(When running headless, it still gets the cloudflare checkbox on login, so I made epic-games ignore SHOW=0 - in Docker everything is running non-headless anyway.)

Note that your $BROWSER_DIR (default data/browser) has all the files created by Firefox in it. For me it was no problem to let Chromium also use it, but it's not a good idea (also just wastes space) and may lead to issues. You can just move/delete it.

Let's track the OpenGL in Docker issue here: #483
The --no-sandbox message (I assume that's what you mean) is due to Chrome now complaining about being run as root (default for Docker). See #468. I already have a PR for it, but it will require everyone to change permissions of the files in their Docker volume (fgc in the command from the readme) if they want to keep using them. If you don't care about starting new data/*.json files (history of what was claimed, including (un)redeemed keys for prime-gaming), you can just use a new volume (e.g. fgc2) and merge the files later if you want.

Not sure what people care about and how to best communicate breaking changes like the above changing permissions for existing Docker volumes.
I think it's best for all to watch new releases on GitHub and I'll mention it there if you need to do something (also in the logs, but not sure how many people check them - another option would be notifications (for people who have NOTIFY set up)):
